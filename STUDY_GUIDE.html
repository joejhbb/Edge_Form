EdgeForm AI Study Manual

Simpleng code explanation at gabay para sa thesis defense sa loob ng 3 araw. 100% C# at React integration.

1. Ang Atong Application (System Overview)

Ang EdgeForm AI ay isang digital posture-coaching system na nagbibigay-daan sa real-time biomechanical analysis habang nag-eehersisyo (tulaby ng Bodyweight Squats at Pushups/Planks).

Taglish Paliwanag:

Ito ay isang web app kung saan ginagamit ang laptop webcam para basahin ang iyong joints gamit ang deep-learning camera algorithms. Sa halip na i-count lang ang squat mo na walang kuwenta, ang program mo ay napakatalino dahil ipinapadala nito ang coordinates sa isang C# ASP.NET Core Web API. Ang C# backend ang nagko-compute ng angles gamit ang Trigonometry para husgahan kung tama o mali ang iyong form!

May dalawang (2) bahagi ang system na ito:

  • Frontend (React + Vite + TypeScript): Gumagana bilang "mga mata" ng system gamit ang video overlay at "boses" nito (TTS voice commands). Ini-isolate nito ang coordinates ng katawan (17 keypoints tulad ng tuhod, balikat, at kurba ng dila).
  • Backend (C# ASP.NET Core API): Gumagana bilang "utak" o taga-husga ng system. Ito ang tagatanggap ng coordinates, taga-compute ng geometric vectors, at taga-counter ng reps batay sa biomechanical ranges.

3. Paano Umaandar ang Frame Loop? (The Pipeline)

Kritikal na maipaliwanag mo sa mga examiners kung papaano dumadaloy ang data kapag may gumagamit sa tapat ng camera.

Unang Hakbang - Pagkuha ng Larawan

Binabasa ng camera ang live video frames ng user sa browser. Hinahati ng TensorFlow.js ang frames at gumagawa ng listahan ng 17 keypoints (X at Y coordinates) na sumasalamin sa mga kasukasuan.

Ikalawang Hakbang - Ang C# API Request

Bawat frame (capped gamit ang state latch para iwas lag), nagpapadala ang React app ng HTTP POST Request sa C# API. Naglalaman ang payload ng exercise type at ang array ng coordinates. Halimbawa:

{
  "exercise": "squat",
  "pose": {
    "keypoints": [ {"name": "left_knee", "x": 120.4, "y": 240.2, "score": 0.98}, ... ]
  },
  "state": { "reps": 4, "isDescending": true }
}
Ikatlong Hakbang - Mathematical Computation

Binabasa ng C# server ang angles, sinusuri ang mga limitasyon (Hysteresis), binibilang ang bagong reps (gamit ang customized state machine), at nagbabalik ng malinis na sagot (JSON Response) na nagsisilbing diagnostic score para sa user.

Ikaapat na Hakbang - Feedback Execution (TTS)

Tatanggapin ng React ang score at text message. Sasabihin nito sa audio engine na magsalita (e.g., "Widen feet to shoulder-width!") kapag may maling naitampok sa workout quality.

5. Deep Dive sa C# Biomechanics (PoseAnalyzer.cs)

Ito ang pusong-buhay ng mathematics ng iyong application. Dito ginagawa ang skeletal evaluation:

CalculateAngle Code Breakdown:

public static double CalculateAngle(Keypoint p1, Keypoint p2, Keypoint p3)
{
    // Math.Atan2 computes the angle in radians of two lines with respect to x-axis
    double radians = Math.Atan2(p3.Y - p2.Y, p3.X - p2.X) - Math.Atan2(p1.Y - p2.Y, p1.X - p2.X);
    double angle = Math.Abs((radians * 180.0) / Math.PI);
    
    if (angle > 180.0)
    {
        angle = 360.0 - angle; // Laging kuhanin ang sharp interior angle (0° - 180°)
    }
    return angle;
}

Taglish Paliwanag (Squat Rep Counting):

Si C# ay gumagamit ng State Machine o dynamic state memory upang maiwasan ang maling rep-counting kapag nanginginig ang katawan:

  1. Kung ang average angle ng iyong tuhod (kneeAngle) ay lumagpas sa 155°, kinokonsidera ito na nakatayo ng tuwid (Neutral Stand). Sa puntong ito, naghahanda ang system.
  2. Kung ang anggulo ay bumaba sa 140°, si IsDescending ay nagse-set sa true (Bumababa na si user).
  3. Upang makakuha ng puntos ang malalim na rep, dapat umabot ang tuhod sa sa porsyentong 90° - 100°. Kapag nagawa ito, ang flag na DepthReached ay magiging true.
  4. Kapag tumayo muli ang user ng tuwid lagpas sa 155°, tinitingnan ni C# kung nag-true ang DepthReached. Kung OOO, magdaragdag ng +1 sa 'Reps' at ire-reset ang tracker para sa susunod na bilang!

7. Mathematical Formulas and Geometry (The Trigonometry)

Isang malaking bahagi ng grado sa defense ay nakasalalay sa pagpapaliwanag mo ng Math functions.

Angle(α) = | Atan2(y3 - y2, x3 - x2) - Atan2(y1 - y2, x1 - x2) | * (180 / π)

Sa coordinate math ng application, bawat joint ay tinuturing na 2D point $(X, Y)$ sa space:

  • p2 ang nagsisilbing vertex o ang sentrong point (hal. ang Tuhod).
  • p1 at p3 ang mga boundary connections (hal. ang Hips / Ankle).
  • Kalkulahin muna ang angle slope ng point 3 relatibo sa point 2 gamit ang Atan2(y3 - y2, x3 - x2).
  • Gawin din ang parehong hakbang para sa point 1 relatibo sa point 2: Atan2(y1 - y2, x1 - x2).
  • Bawasan ang dalawang slope angles upang makuha ang absolute difference, at i-multiply sa $\frac{180}{\pi}$ upang i-convert ang raw Radians na value sa human-readable na Degrees (0° to 180°).

8. Thesis Defense QA (Ready-to-Answer Questions)

Narito ang mga madalas itanong ng panel examiners tungkol sa ganitong software architectures:

Q1: "Bakit C# ang backend nyo at hindi purely React? Ano ang benefit nito?"

Answer: "Sa pamamagitan ng paghihiwalay ng computational mathematics sa isang C# ASP.NET Core API server, hindi natin pinapabigatan ang browser execution thread ng React frontend. Samakatuwid, nagiging mabilis at may mataas na frame rate (smooth 30fps to 60fps) ang visualization sa dynamic screen ng user habang ang solid mathematical tracking at state variables ay tinutuos sa server layer nang ligtas at maaasahan."

Q2: "Paano nadedetek ng camera kung madilim o maliwanag ang silid?"

Answer: "Gumagamit kami ng real-time pixel scanning sa raw feed ng webcam upang masuri ang luminance values. Ang nakukuhang light quality state ay pinapasa sa endpoint ng API. Kapag napansin ng C# server na 'dark' o 'bright' ang setting at bumaba ang keypoint accuracy score, agad na nagbabalik ang API ng visual warning prompt sa screen para gabayan ang user."

Q3: "Ano ang ginagawa ng application kapag may joint detection glitch o mabilisang jitter?"

Answer: "Nag-implement po kami ng hystereis cooldown at moving noise filters. Sa WorkoutView.tsx, hihintayin muna nating lumipas ang 1.5 segundo ng sunod-sunod na signal interruption bago ipasok sa screen ang babala. Ito ang umiwas sa flickering feedback upang manatiling kalmado at propesyonal ang user experience."
